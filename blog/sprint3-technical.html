The differences between HTML and CSS

Imagine HTML and CSS are your good friends. They have aspergers and are really amazing at particular things.
For example, HMTL is great with creating text, images and other content and CSS is a wonder with colours, placement and other styling things and also loves curly braces. Luckily for us, they work really well with each other and are good communicators when it comes to this.
To balance all this amazingness out they are unable to do the things their buddy can do. Sometimes CSS jumps in and adds some colour or styling in HTMLs workbook and that's fine but HTML never writes in CSS's workbook. They both work better when there is order and things are done how they expect them to be done, no new fancy cajun tofu and butter bean dinners for them, just the usual.

At first you thought they were confusing and that they didn't get on well with people very well but once you learnt that they have their special ways of doing things and accepted that this is how they are your friendships bloomed and things became a lot easier, apart from the melt downs that occured for leaving out a semicolon or a bracket, that never got easier.

Control flow and loops

Control flow is kind of like getting dressed - You put on undies first and then pants, never the other way around unless you're being a superhero in which the order goes undies, pants then undies.

Loops are useful for sorting through data and pulling out the information that is asked for. There are 5 different loops which do slightly different things.

`for` - loops through a block of code for a number of times
`for/in` - loops through the properties of an object
`for/of` - loops through the values of an iterable object
`while` - loops through a block of code while a specified condition is true
`do/while` - also loops through a block of code while a specified condition is true

For example, to find out to find out what's in the pantry for breakfast you could use the following `for` statement.


var breakfastOptions = {
"muesli" : 4,
"peanutButter" : 17,
"eggs" : 3,
"honey" : 0
}
for (var i = 0; i < breakfastOptions.length, i++) {
return breakfastOptions[i];
}
console.log(breakfastOptions);

The DOM

The DOM stands for “Document Object Model”.
It is a way to inspect what is going on in a webpage and is quite cool cause you can manipulate the existing code of a website. The DOM comes up on a webpage when opening right clicking and selecting 'inspect'.
`.getElementById()`, `getElementByClassName()` and `getElementByTagName()` allows all elements within the previously specified area (eg document) to call an element by it ID, class or tag name(eg `li` tags) and using `var`can store them using a new keyword for using in the DOM

The difference between accessing data from arrays and objects

The difference is that arrays are like ordered lists and you refer to what number (starting from 0) item you would like to access and objects are like unordered lists, they don't have numbers so you refer to what the name of the key is to access the data.

Accessing data from an array goes like this:
myArray[3] -- this locates the 4th item in the array list.

Accessing an item in an object goes like this:
myObject["pants"] -- this locates the item called "pants" from the object list.

What are functions and why are they useful?

Functions are a block of code that does a particular task when asked to.
They are useful because they make completing tasks faster and more efficient and accurate than writing them down in long form. This makes it easier to read the code and I assume makes the script run faster. Functions are a fundamental part of Javascript
